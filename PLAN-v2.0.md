# OpsPilot v2.0 全面增强计划

## 阶段一：WebSSH 终端 (P0)
**目标：** 后端已有 `terminal.rs` + `ws_events.rs` (WebSocket→SSH 代理)，前端缺 xterm.js 终端组件
**交付：**
- 前端 `TerminalPage` + `WebTerminal` 组件 (xterm.js + websocket)
- App.tsx 添加终端路由 `/terminal/:hostId`
- 侧边栏"基础设施"分组下添加终端入口
- 移动端导航栏添加终端
- i18n 补充 terminal 相关翻译
- TS 零错误 + 测试通过

## 阶段二：操作审计日志 (P0)
**目标：** 后端的 `audit.rs` 已有但无前端页面
**交付：**
- 前端 `AuditLog` 页面：时间范围筛选、操作类型筛选、用户搜索、分页表格
- 后端 audit 路由增强：分页、筛选、导出 CSV
- 侧边栏"系统管理"分组下添加审计入口
- i18n 补充 audit 翻译

## 阶段三：用户/角色权限管理 RBAC (P1)
**目标：** 从单人模式升级到多用户角色管理
**交付：**
- 后端：User Role 模型扩展 (admin/operator/viewer)、权限中间件
- 前端：用户管理页面（列表/创建/编辑/删除）、角色分配
- 登录注册增强：支持角色返回
- 页面级权限控制（viewer 不能操作敏感功能）

## 阶段四：Dashboard 可定制仪表盘 (P0)
**目标：** 静态数据大屏 → 可拖拽添加/删除小部件
**交付：**
- 基于 `react-grid-layout` 的拖拽网格
- 可用小部件池：健康总览、模块状态、快速入口、最近告警、资源使用率
- 布局持久化（localStorage）
- 可添加/移除小部件的配置面板

## 阶段五：告警规则管理与通知中心 (P0)
**目标：** 从仅有告警升级到完整的告警生命周期管理
**交付：**
- 告警规则 CRUD（阈值/条件/静默期/通知渠道）
- 告警历史列表 + 详情
- 通知渠道管理（邮件/Webhook/钉钉/企业微信）
- 告警静默规则

## 阶段六：CMDB 配置管理数据库 (P1)
**目标：** 主机 → 应用/服务/中间件的关联管理
**交付：**
- 服务/应用模型 CRUD
- 服务与主机关联
- 服务依赖关系图
- 配置项版本管理

## 阶段七：批量操作/多选命令 (P2)
**目标：** 多台主机批量执行命令
**交付：**
- 主机多选 → 批量命令执行
- 执行结果聚合展示
- 命令模板保存

## 阶段八：Grafana 风格图表 (P2)
**目标：** 监控数据可视化增强
**交付：**
- 时序图表增强（缩放/区间选择/多指标叠加）
- Prometheus 数据源集成（可选）
- 自定义仪表盘图表

## 阶段九：事件时间线 (P2)
**目标：** 所有事件汇聚到统一时间轴
**交付：**
- 事件汇聚：告警 + 操作 + 状态变更 + 审计
- 时间轴可视化
- 筛选/搜索

## 阶段十：AI 运维助手增强 (P2)
**目标：** AgentChat 增强
**交付：**
- 自然语言→查询（日志/指标/资产）
- 异常自动诊断
- 运维知识 RAG（基于已有 knowledge 模块）

## 阶段十一：OpenAPI/Swagger (P2)
**目标：** 后端 API 自动化文档
**交付：**
- 集成 utoipa/okapi 自动生成 OpenAPI 规范
- 添加 Swagger UI 路由
- Cargo 依赖更新

## 阶段十二：PWA + i18n 完整度 (P2)
**目标：** 移动端体验增强
**交付：**
- PWA manifest + Service Worker
- i18n 完整覆盖检查与补全
- 离线缓存策略
