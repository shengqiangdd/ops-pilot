# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 2.x     | :white_check_mark: |
| < 2.0   | :x:                |

## Reporting a Vulnerability

We take the security of OpsPilot seriously. If you believe you've found a
security vulnerability, please report it by emailing us at **security@opspilot.dev**.

**Please do NOT report security vulnerabilities through public GitHub issues.**

You should receive a response within 48 hours. If for some reason you do not,
please follow up to ensure we received your original message.

Include as much information as possible:

- Type of vulnerability
- Full paths of source files related to the vulnerability
- Step-by-step reproduction instructions
- Proof-of-concept or exploit code (if possible)
- Impact of the vulnerability

## Disclosure Policy

- We will acknowledge receipt of your vulnerability report within 3 business days
- We will send a more detailed response within 10 business days
- We will keep you informed of the progress towards a fix
- We will publicly disclose the issue after a fix has been released
